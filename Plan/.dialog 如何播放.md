# .dialog 如何播放

## 问题本质

`.dialog` 是一个**数据容器**，不是**执行器**。

```
DialogSequenceSO (数据)
    ├── Entries[0]: DialogEntry { Speaker="Alice", Content="你好" }
    ├── Entries[1]: AvatarEffectEntry { CharacterId="Alice" }
    └── Entries[2]: DialogEntry { Speaker="Bob", Content="你好，Alice" }
```

问题：这些数据如何变成屏幕上逐行显示的文字？

---

## 两种播放模式

### 模式一：Handler 一次性分发（推模式）

```
VFSNode (intro.dialog)
    ↓ 信号触发
Handle() 被调用
    ↓ 遍历 Entries
PostSystem.Send("Dialog.Show", entry[0])
PostSystem.Send("Avatar.Show", entry[1])
PostSystem.Send("Dialog.Show", entry[2])
    ↓ 立即返回
Handler 结束
```

**特点**：
- Handler 只负责"发事件"，不管"怎么显示"
- UI 层收到事件后自行管理显示时机
- 适合：简单对话、不需要逐句等待

**问题**：
- 如果 Entries 有 10 条，是一次发 10 个事件，还是等 UI 逐条确认？
- 谁来控制"下一句"的触发时机？

---

### 模式二：Handler 阻塞等待（拉模式）

```
Handle() 被调用
    ↓
取出 Entries[0]，发送事件
    ↓
等待 UI 完成信号（怎么等？）
    ↓
取出 Entries[1]，发送事件
    ↓
...
所有 Entries 处理完毕
    ↓
Handler 返回
```

**特点**：
- Handler 控制节奏，一句一句来
- UI 完成后通知 Handler 继续
- 适合：需要逐句确认、打字机效果、等待用户点击

**问题**：
- Handler 是同步执行的，如何"等待"异步的 UI？
- 阻塞 GraphRunner 是否可行？

---

## 核心矛盾

| 矛盾 | 说明 |
|------|------|
| **同步 vs 异步** | Handle() 是同步调用，但对话播放是异步过程（需要时间） |
| **推 vs 拉** | 是一次性发完所有事件，还是等 UI 逐条拉取？ |
| **阻塞 vs 非阻塞** | 如果等待 UI，Handler 不能立即返回，GraphRunner 怎么办？ |

---

## 可能的解决方案

### 方案 A：Handler 只启动，不等待

```csharp
Handle(...)
{
    var sequence = content.GetUnityObject<DialogSequenceSO>();
    // 启动一个对话播放器，不等待
    DialogPlayer.Start(sequence);
    // 立即返回，对话在后台播放
}
```

**问题**：
- 下游节点如何知道"对话播放完了"？
- 如果 VFS 图有分支，对话还没播完就走到判断节点怎么办？

---

### 方案 B：Handler 阻塞，但用状态机

```csharp
Handle(...)
{
    var sequence = content.GetUnityObject<DialogSequenceSO>();
    
    // 设置当前播放状态
    runner.SetState("DialogPlaying", sequence);
    
    // 发送第一条
    PostSystem.Send("Dialog.Start", sequence);
    
    // ??? 如何等待 ???
}
```

**问题**：
- Handle 返回后，谁发第二条？
- 需要外部机制（Update 轮询？事件回调？）来推进播放

---

### 方案 C：把 Sequence 展开成 VFS 图

```
原图:  [Start] → [intro.dialog] → [CheckFlag]

展开:  [Start] → [intro_001] → [intro_avatar] → [intro_002] → [CheckFlag]
              (DialogEntry)   (AvatarEffect)   (DialogEntry)
```

**特点**：
- 每个 Entry 变成一个独立节点
- 利用 VFS 图本身的连接控制流程
- 不需要 Handle 里遍历 Entries

**问题**：
- 需要编辑器支持或代码生成
- 失去 Sequence 作为数据容器的意义

## 待决策的问题

1. **播放节奏由谁控制？**
   - Handler 一次性发完所有事件？
   - 还是 Handler 控制，逐条等待？

2. **如何表示"播放中"状态？**
   - GraphRunner 暂停？
   - 还是继续跑图，但对话在后台播放？

3. **用户点击"继续"如何反馈？**
   - 发送 PostSystem 事件？
   - 还是修改某个共享状态？

4. **分支判断时，对话播完了吗？**
   - 如果对话还没播完就走到 ConditionNode，判断依据是什么？
